# ThirdParty Recipe

## Add Submodule:googletest
```bash
cd path/to/SparseEnd2End
git submodule add https://github.com/google/googletest.git onboard/third_party/googletest
git commit - m "add submodule googletest"
cd  onboard/third_party/googletest
cmake -B build -S .
cmake --build build
```

## Add Submodule:yaml-cpp
```bash
cd path/to/SparseEnd2End
git submodule add https://gitlab.com/libeigen/eigen.git onboard/third_party/eigen
cd onboard/third_party/eigen
git checkout 3.4
cd -
git commit - m "Add Eigen submodule at version 3.4"
cd  onboard/third_party/eigen
cmake -B build -S .
```
> Default installation of Eigen is provided to users in the form of source code(that means you may not use `make`, but use `make install`),  your code just need to include Eigen's header file. The reason is that Eigen is implemented in the form of templates, the template functions do not support separate compilation, it can only provide source code instead of dynamic/static libraries for users to use.

if you want to use `eigen`, you need to build in CMaleLists.txt
```bash
# Method1 absolute
include_directories("/usr/local/include/eigen3")

# Method2 `find_package()`, but you need to ensure ${EIGEN3_INCLUDE_DIR} contain eigen files. Otherwise, you will meet error `fatal error: Eigen/Dense: No such file or directory`.
if((Eigen3_FOUND) AND (DEFINED EIGEN3_INCLUDE_DIR))
  message(STATUS "Found Eigen3: " ${EIGEN3_INCLUDE_DIR})
  include_directories(${EIGEN3_INCLUDE_DIR})
else()
  message(FATAL_ERROR "EIGEN3 not found.")
endif()
```
Or, you may directly run it in comman line with g++:
```bash
g++ test.cpp -o test.o -I /usr/local/include/eigen3
```
